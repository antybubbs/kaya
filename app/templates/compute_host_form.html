How Docker Agent works

Kaya does not need a Docker API URL for this option. Save this connection first, then Kaya will generate a private agent token and a ready-to-run docker-compose file for the Docker server.

  1. Enter a friendly name for the Docker host.
  2. Save the connection.
  3. Copy the generated docker-compose file from the next page onto the Docker server.
  4. Start the agent with docker compose up -d.

The agent calls back to Kaya over HTTP(S), so you do not need to expose Docker ports 2375 or 2376.
