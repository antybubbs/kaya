# Architecture

**Kaya version:** `dev`  
**Documentation version:** `dev`

Kaya is a FastAPI application using server-rendered Jinja templates, SQLAlchemy models, SQLite by default, and local JavaScript/CSS assets.

## Technology Stack

- Python 3.12
- FastAPI
- Starlette sessions
- Jinja2 templates
- SQLAlchemy ORM
- SQLite default database
- Passlib Argon2 password hashing
- Fernet encryption for stored secrets
- Pandas/OpenPyXL for import/export support
- dnspython and urllib-based domain lookups
- Node.js helper services for SSH and Guacamole/RDP bridging
- `guacamole-lite`, `ssh2`, and `ws` Node packages
- Docker Compose for deployment

## Folder Structure

- `app/main.py`: FastAPI app setup, middleware, router registration, startup/shutdown, database bootstrap/migrations
- `app/models/models.py`: SQLAlchemy models
- `app/db/session.py`: database engine and session factory
- `app/core/`: configuration, security, CSRF, demo mode, branding, TOTP helpers
- `app/routers/`: route modules for each feature area
- `app/services/`: shared business logic, background polling, integrations, audit/session helpers
- `app/templates/`: Jinja templates
- `app/static/`: CSS, JavaScript, brand assets, vendor assets
- `scripts/`: migration, demo seeding, Node remote helper services
- `tests/`: currently limited test coverage, mainly demo-mode restrictions

## Main Entry Points

- `app.main:app` for Uvicorn
- `/setup` for first admin creation
- `/login` for authentication
- `/dashboard` for the authenticated landing page
- `/healthz` for health checks

## Routing

Routing is modular but centrally registered in `app/main.py`. Routers are included for:

- Auth
- Dashboard
- Licences
- IP/VLANs
- Hardware assets
- Network monitor
- Remote manager
- Runbooks
- Domain manager
- Compute manager
- Rack manager
- Backup manager
- DNS manager
- Admin/system pages

## Templates And Static Assets

Templates are server-rendered from `app/templates`. `base.html` provides the main navigation, header, account menu, theme toggle, sidebar, and shared static asset includes.

Static assets are local under `app/static`. The main CSS files are:

- `app.css`
- `imports.css`
- `demo.css`
- `kaya.css`

Shared JavaScript includes sidebar state, table sorting/filtering/column settings, and inline editing. Module-specific JavaScript lives in `app/static/js`.

## Configuration

Configuration is split between environment variables and database-backed settings.

Environment settings are defined in `app/core/config.py`. Runtime secrets are generated by `docker-entrypoint.sh` into `/app/data/.runtime.env` if not supplied.

Database-backed site settings, remote settings, backup settings, SMTP settings, security settings, and several module options are stored in `remote_manager_settings`.

Defaults are defined in:

- `app/services/site_settings.py`
- `app/routers/remote_manager.py`

## Background Services

The FastAPI process starts background work on application startup:

- Network monitor loop
- Domain polling loop
- Compute monitor loop
- Kaya remote SSH helper service
- Guacamole bridge service when enabled

These workers run in-process. Multi-instance deployments can duplicate work unless this architecture is changed.
